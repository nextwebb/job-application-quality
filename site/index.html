Why this exists

The missing quality gate between generation and submission.

Most job-search agents optimize for volume. JAQ optimizes for trust: every packet gets checked against canonical facts, role requirements, policy constraints, and approval state.

AI job workflow

  1. Finds role
  2. Drafts CV
  3. Drafts cover letter
  4. Prepares packet

job-application-quality

  1. Checks truth
  2. Checks policy fit
  3. Checks artifact quality
  4. Creates manifest
  5. Requires approval

Human

  1. Reviews
  2. Approves
  3. Submits

Quality gates

The system fails loudly when a packet should not move forward.

STOP

Hard fail

  • Unsupported candidate claim
  • Sponsorship or work authorization mismatch
  • Remote or location mismatch
  • Final submit without explicit approval
WARN

Review warning

  • Required skill lacks supporting evidence
  • Evidence is truthful but weak
  • Optional artifact is missing
  • Role fit needs human judgment
PASS

Review-ready

  • Claims map to canonical facts
  • Policy settings match role constraints
  • Manifest records exact inputs and outputs
  • Approval checkpoint remains explicit

Architecture

Small contracts. Deterministic checks. Human approval.

  1. Intake Load candidate profile and role JSON.
  2. Validate Run structural checks for facts, ATS, location, and requirements.
  3. Gate Block unsupported sponsorship, remote, ATS, or authorization paths.
  4. QA Inspect generated artifacts for placeholders, path leaks, and hallucinated claims.
  5. Manifest Record hashes, selected files, policy result, and approval state.
manifest.json 
{
  "tenant_id": "candidate_a",
  "role_id": "backend-platform-role",
  "policy_gates": { "status": "pass" },
  "outputs": [
    { "type": "cv", "qa_status": "pending" }
  ],
  "approval": {
    "required": true,
    "status": "pending"
  }
}

CLI first

One command path for humans, agents, and CI.

JAQ is dependency-light and local-first. The public CLI wraps the same deterministic Python scripts that agents use under the hood. 

git clone https://github.com/nextwebb/job-application-quality.git
cd job-application-quality

./bin/jobqa demo
./bin/jobqa init /tmp/my-application
./bin/jobqa run /tmp/my-application
./bin/jobqa report /tmp/my-application/output/manifest.json --format markdown
