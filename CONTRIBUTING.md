# Contributing

This project is intentionally conservative. It is for high-quality application preparation, not uncontrolled mass submission.

## Contribution Rules

- Open an issue before major changes.
- Do not commit real resumes, tenant profiles, salary details, immigration documents, or application logs.
- Keep the core local-first and file-based.
- Keep email send and browser final-submit paths approval-gated.
- Add tests for policy gates and unsupported claim detection.
- Update docs when behavior or schema changes.

## Pull Request Checklist

- [ ] Scripts pass on a fresh clone.
- [ ] No real personal data is committed.
- [ ] Schema examples still validate.
- [ ] Hard gates fail closed.
- [ ] Release notes updated for user-visible behavior.
